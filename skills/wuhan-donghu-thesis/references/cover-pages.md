# 武汉东湖学院毕业论文封面前两页可执行规范

本文件用于约束 agent 生成论文前两页。只要生成或修改武汉东湖学院本科毕业论文 Word 文档，就必须优先执行本规范，再进入摘要、目录和正文排版。

## 1. 适用范围与基本原则

1. 前两页固定为中文封面和英文封面，不属于普通正文。
2. 前两页不得套用正文段落样式，不得统一设置为正文的固定 23 磅行距、首行缩进 2 字符或段后 10 磅。
3. 中文封面必须包含校徽和“武汉东湖学院 / Wuhan Donghu College”校名字样，不允许只用普通文字代替。
4. 若存在 `范本.docx`，最稳妥的做法是复制范本前两页的 `FormattedText`，只替换字段文本，不重建版面。
5. 若只有本指南可用，必须使用本文件的资产、段落序列和验收规则重建前两页。

## 4.1 右上角学号与档号硬规则

范本右上角不是普通右对齐文字，而是带下划线运行和制表位的两行 Word 对象。生成或替换时必须满足：

| 项目 | 范本参数 |
|---|---|
| 段落位置 | 中文封面第 2 段为学号，第 3 段为档号，档号必须紧跟学号 |
| 段落对齐 | 两段均为右对齐 |
| 字体字号 | 标签“学号：”“档号：”为黑体五号；数字、字母、连字符和占位空格为五号并加单下划线 |
| 制表位 | 学号段包含 `8280` twips 左制表位；档号段包含 `7920` 与 `8280` twips 左制表位 |
| 固定槽宽 | 学号和档号冒号后的下划线区域必须使用同一固定显示宽度；无论值长短，右端必须对齐 |
| 学号行文本结构 | `学号：{居中填入学号的固定宽度下划线槽}`；冒号后的空格、学号和尾随空格必须在带下划线的 run 中，空格必须保留 |
| 档号行文本结构 | `档号：{居中或左起填入档号的固定宽度下划线槽}`；若无档号，写入同宽度带下划线空格占位，不得删除横线 |

固定槽宽不是“字符串长度够长即可”，也不能按“中文算 2、英文算 1”的字符宽度估算。Word 中数字、中文、普通空格和不同字体的实际渲染宽度不同，必须以 `范本.docx` 的 Word 渲染坐标为准：`学号/档号` 为一组，`院（系）名称/专业名称/学生姓名/指导教师` 为一组。生成时应将实际值放入槽内并补带下划线空格，使每一行的标签起点、冒号后下划线起点、下划线右端与范本对应段落保持一致；短值不能让整行右移或横线变短。

OpenXML 生成时，冒号后的空格必须使用 `<w:t xml:space="preserve">`，并在同一 run 或相邻 run 的 `<w:rPr>` 中设置 `<w:u w:val="single"/>`。在有 `范本.docx` 且需要模板级对齐时，不得用普通文本下划线、边框线、图片线或删除空格来模拟。

如果无法稳定重建这些参数，必须改用 `scripts/copy_cover_from_sample.ps1` 从 `范本.docx` 复制前两页，只替换字段文本。

### 4.1.1 无范本时的表格固定槽备用方案

当没有 `范本.docx`，或目标平台无法稳定保留 run 级尾随空格时，可以把表格固定槽作为从零重建封面的备用方案。该方案不是模板复制的替代品；只在无法复制范本时使用，并且必须通过渲染截图验收。

实现原则：

1. 右上角学号/档号可用 2 行 2 列无框表格实现：第 1 列放标签，第 2 列为固定宽度值槽。
2. 院（系）名称、专业名称、学生姓名、指导教师可用 4 行 2 列无框表格实现：第 1 列放标签，第 2 列为固定宽度值槽。
3. 值槽单元格设置固定列宽、内容居中、垂直靠下、左右内边距为 0，仅保留下边框 `single`；其他单元格边框必须为 `none`。
4. 表格整体应按范本位置右对齐或居中，值槽左端、右端和文本中心必须与范本截图或坐标目标一致；不能只依赖字符数估算。
5. 生成后必须运行 OpenXML 校验。`w:rPr`、`w:tblPr`、`w:tcPr` 内子元素顺序必须符合 schema；如果工具生成的 `rFonts`、`sz`、`tcBorders` 顺序错误，必须后处理修正。
6. 必须用 Word/PDF 渲染结果验收。开启格式标记时，表格方案不应出现为了撑宽而产生的大量空格点；打印视图中只能看到等长横线和居中文本。

注意：表格方案会改变封面 Word 对象结构，通常无法通过以 `范本.docx` 段落序号、制表位、run 级下划线为依据的 `verify_cover_against_sample.py`。只要有范本，优先复制范本并用渲染坐标校准；表格方案只作为无范本环境的可复现 fallback。

PowerShell 脚本包含中文字符串时，必须使用 PowerShell 7 `pwsh`，或保存为 UTF-8 with BOM。Windows PowerShell 5 可能把 UTF-8 无 BOM 脚本中的 `学号：`、`档号：`、`黑体` 读成乱码，严重时会导致脚本语法错误。

## 7. 推荐自动化流程

### 7.1 有范本时

1. 打开 `范本.docx`。
2. 取第 1 页起点到第 3 页起点之前的 `FormattedText`。
3. 删除目标论文原有前两页。
4. 将范本前两页插入目标论文文首。
5. 只替换字段文本，不改变段落数、分页符、图片、锚点和直接格式。
6. 复制后重新应用前 47 个段落的直接段落格式，避免被目标文档 `Normal` 样式污染。
7. 复制后必须从 `范本.docx` 回填前 47 段的实际字体属性，包括 `Name`、`NameAscii`、`NameFarEast`、字号和粗体状态。否则目标文档主题可能把中文污染成 `MS Mincho/+中文正文`，把英文污染成 `Cambria/+西文正文`。
8. 对第 2、3、15、16、17、18 段使用 Word 渲染坐标校准：读取范本对应段落的行起点、冒号后下划线起点和下划线右端，给候选文档补带下划线空格，直到坐标误差在 2.5 pt 内。
9. 保存后重新分页并渲染验收。

该流程可用 Word COM 脚本实现。脚本原理、中文路径安全处理和复用边界见 `自动化脚本原理与复用规范.md`。

### 7.2 无范本时

1. 按本文件第 5、6 节创建 47 个封面段落。
2. 插入 `wdu_cover_logo.png` 和 `wdu_cover_calligraphy.png`，使用浮动或等效稳定布局，锚点分别放在第 1 段和第 5 段。
3. 第 24 段和第 47 段必须包含分页符。
4. 生成声明页时从第 3 页开始。
5. 渲染前两页，与参考图逐项核对。

## 8. 必须执行的验收检查

生成 Word 后必须执行以下检查：

1. 导出前两页为 PDF 或 PNG。
2. 检查第 1 页是否同时有校徽、校名字样、中文标题、院系专业姓名教师和中文日期。
3. 检查第 2 页是否只有英文封面内容，不得出现中文日期或声明页内容。
4. 检查第 3 页是否开始显示独创性声明。
5. 检查第 1 页中文日期所在页码必须为 1。
6. 检查英文封面标题所在页码必须为 2。
7. 检查前两页无页眉、无页码。
8. 检查文档 XML 或 Word 校验无错误。
9. 运行 `scripts/verify_cover_identifiers_and_length.py --skip-length-check`，确认学号/档号右上角结构、制表位和下划线未丢失。
10. 若有 `范本.docx`，再运行 `scripts/verify_cover_against_sample.py --sample 范本.docx --candidate 论文初稿.docx`，动态比对范本中的学号/档号段落序号、右对齐、制表位、标签字体字号、冒号后字号、下划线 run 和空格保留。
11. 在 Windows + Word 环境中运行 `scripts/verify_cover_rendered_alignment.ps1 -SamplePath 范本.docx -CandidatePath 论文初稿.docx`，确认第 2、3、15、16、17、18 段的行起点、下划线起点、下划线右端与范本渲染坐标一致。
12. 在 Windows + Word 环境中运行 `scripts/verify_cover_fonts_against_sample.ps1 -SamplePath 范本.docx -CandidatePath 论文初稿.docx`，确认封面关键段落的实际字体没有被目标文档主题替换。

第 11 项是固定槽宽的最终判据：`学号/档号` 必须左右端对齐；`院（系）名称/专业名称/学生姓名/指导教师` 的下划线右端必须与范本一致，短值不能让横线变短，长值不能让右端越界。

如果任一项失败，不得交付文档，必须回到封面生成步骤修正。

## 9. 常见失败原因

| 失败表现 | 根因 | 修正 |
|---|---|---|
| 没有校徽或校名字样 | agent 把封面当普通文本重建 | 使用 `cover_assets` 资产或复制范本前两页 |
| 中文日期单独跑到第 2 页 | 封面段落继承目标文档段后 10 磅或正文样式 | 前 47 段段前段后按表格重设，尤其段后为 0 |
| 英文封面变成第 3 页 | 中文日期已溢出到第 2 页，分页符继续推后一页 | 先修正中文封面分页 |
| 封面横线错位 | 删除了模板中的空格或下划线位置 | 只替换字段，不删除整行结构 |
| 专业、姓名或教师值较短时横线变短 | 只替换字段文本，或按字符宽度补空格，没有按 Word 实际渲染宽度校准 | 按范本渲染坐标补带下划线空格，并运行 `verify_cover_rendered_alignment.ps1` |
| 右上角“档号：”后没有横线 | agent 把档号当成可省略字段 | 档号未知也要写入带单下划线的空格占位，并保留制表位 |
| 学号数字没有横线 | 冒号后的数字和空格未设置 run 级下划线 | 对冒号后的所有 run 设置 `<w:u w:val="single"/>`，并用 `xml:space="preserve"` 保留空格 |
