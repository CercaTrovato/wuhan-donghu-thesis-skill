# 武汉东湖学院本科毕业论文 Word 格式与排版规则指南（Codex 执行版）

## 0. 使用定位

本指南供 Codex / AI 在生成、排版、修改武汉东湖学院本科生毕业论文（设计）Word 初稿时执行。本文档不是学生写作教材，也不是某一篇论文的写作方案。

本指南适用于武汉东湖学院本科毕业论文类 `.docx` 文档，包括但不限于初稿生成、格式统一、图表排版、参考文献整理、目录页码更新、送审稿与定稿前检查。

执行本指南时，重点是 Word 文档格式、段落参数、页眉页码、图表对象、表格边框、引用格式和最终验收。内容写作要求仅保留必要的通用约束。

## 1. 依据文件与优先级

### 1.1 必须读取的资料

Codex 在生成或修改论文 Word 初稿前，必须先读取或确认以下资料：

| 优先级 | 资料 | 用途 |
|---|---|---|
| 1 | `论文格式.doc` | 学校正式格式要求，作为最高依据 |
| 2 | `论文初稿写法及要求.docx` | 学校初稿阶段的补充要求和老师经验规则 |
| 3 | `范本.docx` | Word 示例论文模板，用于核对版面、封面、声明、摘要、目录、正文、参考文献等呈现方式 |
| 4 | `论文生成总流程清单.md` | 用于指导 agent 从任务书、开题报告、源码、截图和图表参考中组织论文内容 |
| 5 | `封面前两页可执行规范.md` | 用于生成和验收中文封面、英文封面，尤其是校徽、校名字样和分页 |
| 6 | `目录页可执行规范.md` | 用于生成和验收目录页，尤其是 TOC 样式、制表位、点线前导符和三级标题层级 |
| 7 | `自动化脚本原理与复用规范.md` 与 `scripts` 文件夹 | 用于理解和复用 Word COM、OpenXML、封面复制、目录更新和验证脚本 |
| 8 | `论文各种图表的样式参考` 文件夹 | 用于核对 E-R 图、实体属性图、模块图、流程图、用例图、表格样式 |
| 9 | `武汉东湖学院毕业论文修改指南` | 用于补充已有实操经验，尤其是图片、表格、参考文献和验证流程的高风险问题 |

### 1.2 判断优先级

1. 学校正式格式要求优先于通用论文经验。
2. 学校正式格式要求优先于 Word 示例模板。
3. 示例模板用于补充学校文件没有写细的版面细节。
4. 已有修改指南用于补充自动化处理时容易出错的技术细节。
5. 如果学校文件、模板、已有经验规则发生冲突，优先按学校正式要求处理，并在修改说明中写明冲突点。

### 1.3 禁止默认假设

- 不得用“常见本科论文格式”替代武汉东湖学院实际要求。
- 不得因为某个模板看起来不同，就覆盖学校正式要求。
- 不得把某一篇成稿中的偶然格式当成全校通用要求。
- 不得把具体论文题目、系统功能或技术栈写入本通用指南。

## 2. 论文整体 Word 结构

### 2.1 标准组成顺序

论文 Word 文档应按以下顺序组织：

1. 中文封面
2. 英文封面
3. 独创性声明
4. 学位论文版权使用授权书
5. 中文题目、中文摘要、关键词
6. 英文题目、ABSTRACT、Key words
7. 目录
8. 正文
9. 参考文献
10. 科研情况目录（可选）
11. 附录（可选）
12. 致谢

### 2.2 分页与分节规则

| 位置 | 操作要求 |
|---|---|
| 中文封面后 | 使用分页符或分节符进入英文封面 |
| 英文封面后 | 使用分页符或分节符进入声明页 |
| 声明页后 | 使用分页符或分节符进入中文摘要 |
| 中文摘要后 | 使用分页符或分节符进入英文摘要 |
| 英文摘要后 | 使用分页符或分节符进入目录 |
| 目录后 | 使用分节符进入正文，并从正文开始设置页眉和阿拉伯页码 |
| 正文章与章之间 | 使用分页符，不用连续空行硬撑 |
| 正文结束后 | 使用分页符进入参考文献 |
| 参考文献、附录、致谢之间 | 使用分页符分隔 |

### 2.3 分页禁止事项

- 不允许通过大量空行制造新页。
- 不允许某页开头出现大段空白。
- 不允许图和图题、表和表题拆到两页。
- 不允许一段文字的第一行单独留在上一页或最后一行单独落在下一页。
- 不允许目录页、摘要页误带正文页眉。

## 3. 页面设置

### 3.1 页面尺寸与页边距

| 项目 | 参数 |
|---|---|
| 纸张 | A4 |
| 上边距 | 25 mm |
| 下边距 | 20 mm |
| 左边距 | 30 mm |
| 右边距 | 30 mm |
| 页眉距边界 | 15 mm |
| 页脚距边界 | 15 mm |

### 3.2 OpenXML / 自动化换算参考

| 项目 | 约等值 |
|---|---|
| 1 inch | 1440 twips |
| 1 cm | 约 567 twips |
| A4 宽度 | 11906 twips |
| A4 高度 | 16838 twips |
| 25 mm | 约 1417 twips |
| 20 mm | 约 1134 twips |
| 30 mm | 约 1701 twips |
| 15 mm | 约 850 twips |

Codex 使用脚本设置页面时，应显式写入纸张和页边距，不依赖 Word 默认值。

### 3.4 文档网格

范本第 3 页到摘要页使用 Word 文档网格控制中文段落的实际行高和换行，生成脚本必须显式写入：

| OpenXML 项 | 参数 |
|---|---|
| `w:docGrid/@w:type` | `linesAndChars` |
| `w:docGrid/@w:linePitch` | `312` |

如果缺少该项，或生成成 `linePitch="360"`，即使段落显示为 1.5 倍行距，声明页和授权书也会比范本明显压缩。

### 3.3 页面设置验收

- 所有节均应保持 A4 页面。
- 正文、参考文献、致谢等主体部分应继承相同页边距。
- 不得出现某一节页边距被 Word 默认值覆盖。
- 如果封面模板有特殊排版，应保留模板布局，但不得影响后续正文节。

## 4. 页眉、页脚与页码

### 4.1 页眉规则

| 位置 | 页眉要求 |
|---|---|
| 中文封面 | 无正文页眉 |
| 英文封面 | 无正文页眉 |
| 声明页 | 通常无正文页眉 |
| 中文摘要 | 无正文页眉 |
| 英文摘要 | 无正文页眉 |
| 目录 | 无正文页眉 |
| 正文从“绪论”开始 | 设置页眉 |
| 参考文献 | 继续正文页眉 |
| 附录、科研情况目录 | 继续正文页眉，除非模板另有要求 |
| 致谢 | 继续正文页眉，直到致谢结束 |

页眉内容统一为：

`武汉东湖学院本科生毕业论文（设计）`

页眉格式：

| 项目 | 参数 |
|---|---|
| 中文字体 | 宋体 |
| 字号 | 小五号，约 9 pt |
| 对齐 | 居中 |
| 页眉距边界 | 15 mm |

### 4.2 页码规则

| 位置 | 页码要求 |
|---|---|
| 中文封面 | 不显示页码 |
| 英文封面 | 不显示页码 |
| 声明页 | 通常不显示页码 |
| 中文摘要 | 开始编页码，使用大写罗马数字 |
| 英文摘要 | 延续大写罗马数字 |
| 目录 | 延续大写罗马数字 |
| 正文“1 绪论” | 阿拉伯数字从 1 重新开始 |
| 参考文献 | 延续正文阿拉伯页码 |
| 附录、科研情况目录、致谢 | 延续正文阿拉伯页码 |

页码格式：

| 项目 | 参数 |
|---|---|
| 位置 | 页面底部居中 |
| 字体 | 宋体 |
| 字号 | 小五号，约 9 pt |
| 摘要和目录编号 | 大写罗马数字，如 Ⅰ、Ⅱ、Ⅲ |
| 正文编号 | 阿拉伯数字，如 1、2、3 |

### 4.3 自动化处理注意

- 摘要、目录、正文通常需要不同节，以便设置不同页码格式。
- 正文节必须设置 `restart page numbering at 1`。
- 不要把页码写成静态文本，应使用 Word 页码域。
- 使用 WPS 或 LibreOffice 检查时，如果域显示异常，还必须用 Word 或 OpenXML 检查确认域代码存在。

## 6. 标题层级格式参数

### 6.1 标题参数总表

| 层级 | 编号样式 | 字体字号 | 对齐 | 段前 | 段后 | 行距 | 缩进 |
|---|---|---|---|---|---|---|---|
| 章标题 | `1 标题` | 黑体小二号，18 pt；数字英文 Times New Roman | 居中 | 0.8 行，约 18.4 pt | 0.5 行，约 11.5 pt | 固定 23 磅 | 无首行缩进 |
| 一级节标题 | `1.1 标题` | 黑体四号，14 pt；数字英文 Times New Roman | 顶格左对齐 | 0.5 行，约 11.5 pt | 0.5 行，约 11.5 pt | 固定 23 磅 | 无首行缩进 |
| 二级节标题 | `1.1.1 标题` | 黑体小四号，12 pt；数字英文 Times New Roman | 顶格左对齐 | 0 | 0 | 固定 23 磅 | 无首行缩进 |
| 三级标题 | `（1）标题` | 黑体小四号，12 pt；数字英文 Times New Roman | 左对齐 | 0 | 0 | 固定 23 磅 | 首行缩进 2 字符 |
| 四级标题 | `①标题` | 宋体小四号，12 pt；数字英文 Times New Roman | 左对齐 | 0 | 0 | 固定 23 磅 | 首行缩进 2 字符 |
| 正文 | 正文段落 | 宋体小四号，12 pt；数字英文 Times New Roman | 两端对齐 | 0 | 0 | 固定 23 磅 | 首行缩进 2 字符 |

### 6.2 标题编号要求

- 章标题编号使用阿拉伯数字，如 `1 绪论`。
- 一级节标题使用 `1.1`。
- 二级节标题使用 `1.1.1`。
- 三级标题使用全角括号，如 `（1）`。
- 四级标题使用圈号，如 `①`。
- 编号与标题之间保留一个空格。
- 标题编号必须与目录层级一致。

### 6.3 标题自动化注意

- 可使用 Word 标题样式管理目录，但样式显示必须符合学校参数。
- 如果自动编号导致格式错乱，允许使用手动编号，但必须保证目录、正文标题和页码一致。
- 不得混用 `图3.1`、`图3-1` 或 `表3.1`、`表3-1` 等编号风格。
- 章与章之间使用分页符，不得给章标题设置不受控的 `pageBreakBefore` 后又叠加多余空行。

## 7. 封面与声明页格式规则

### 7.0 封面前两页模板级规则

中文封面和英文封面必须按模板级对象处理，不能按普通正文段落重新排版。生成或修改论文前两页时，必须同时执行本目录下的 `封面前两页可执行规范.md`，并使用 `cover_assets` 文件夹中的校徽、校名字样和参考图进行核对。

封面前两页的优先策略：

1. 如果有 `范本.docx`，优先直接复制范本前两页的完整格式内容，只替换学号、题目、院系、专业、姓名、指导教师、日期等字段。
2. 如果只有本指南，按 `封面前两页可执行规范.md` 中的 47 个段落序列和图片资产重建。
3. 复制或重建后，前两页不得继承正文的 `Normal` 样式、固定 23 磅行距、首行缩进 2 字符或段后 10 磅。
4. 必须渲染前两页检查：中文日期在第 1 页底部，英文封面从第 2 页开始，独创性声明从第 3 页开始。
5. 前两页不得显示正文页眉和正文页码。

如果渲染结果出现“中文日期单独成页”“英文封面变成第 3 页”“缺少校徽或校名字样”等情况，视为封面生成失败，不得交付。

### 7.1 中文封面

中文封面应优先从学校模板复制或沿用。生成初稿时至少保证以下项目完整：

| 项目 | 格式要求 |
|---|---|
| 学号、档号 | 位于中文封面右上角第 2、3 段；右对齐；标签为黑体五号；冒号后的数字、字母、连字符和占位空格为五号并加单下划线；学号段保留 `8280` twips 制表位，档号段保留 `7920` 与 `8280` twips 制表位 |
| “本科生毕业论文（设计）” | 宋体一号加粗，居中，固定 32 磅行距 |
| 中文论文题目 | 中文黑体二号，不加粗，居中；数字和字母 Times New Roman 二号 |
| 副标题 | 中文黑体三号，不加粗，右对齐；数字和字母 Times New Roman 三号 |
| 院系、专业、学生姓名、指导教师 | 中文宋体小三号，数字和字母 Times New Roman 小三号 |
| 日期 | 中文宋体小三号，数字和字母 Times New Roman 小三号 |

封面生成禁止事项：

- 不得改动学校上报题目。
- 不得删除模板要求保留的横线位置。
- 不得省略右上角档号行；档号未知时也必须保留冒号后的带下划线空格占位。
- 不得保留模板红字说明。
- 不得把封面页眉页码带到摘要或正文节。

### 7.2 英文封面

| 项目 | 格式要求 |
|---|---|
| `BACHELOR'S DEGREE THESIS OF WUHAN DONGHU UNIVERSITY` | Times New Roman 二号，加粗，居中，固定 32 磅行距 |
| 英文题目 | Times New Roman 二号，加粗，居中，固定 32 磅行距 |
| Candidate / Supervisor | Times New Roman 小三号，固定 32 磅行距 |
| 英文日期 | Times New Roman 三号，居中 |

英文封面规则：

- 英文题目一般不宜超过 12 个实词。
- 英文题目使用标题式大小写。
- 专有名词按官方写法，例如 MySQL、Vue.js、Spring Boot。
- 英文姓名写法为姓在前、名在后，首字母大写，例如 `Zhang Sansi`。
- 英文日期可写为 `May 8th, 2022` 或 `Apr. 8th, 2022`。月份缩写后加点，未缩写不加点。

### 7.3 独创性声明与授权书

| 项目 | 格式要求 |
|---|---|
| 页序 | 独创性声明和学位论文版权使用授权书必须同在第 3 页 |
| 独创性声明标题 | 宋体二号加粗，居中，固定 23 磅行距，段前段后 0 |
| 声明正文 | 宋体四号，首行缩进约 2 个中文字符，1.5 倍行距或按范本约 18 磅行距 |
| 声明签名、日期区域 | 宋体四号，1.5 倍行距，按范本保留靠右位置 |
| 授权书标题 | 宋体二号加粗，居中，按范本保留段前段后约 7.8 磅 |
| 授权书正文 | 宋体四号，首行缩进约 2 个中文字符，1.5 倍行距或按范本约 18 磅行距 |
| 授权书固定项 | 必须保留“保密口”“本论文属于”“不保密”“学位论文作者签名”“指导教师签名”和双方日期 |

范本关键段落参数：

| 段落 | 行距规则 | 行距值 | 段前 | 段后 | 首行缩进 |
|---|---:|---:|---:|---:|---:|
| 独创性声明标题 | 固定值 | 23 磅 | 0 | 0 | 0 |
| 声明正文 | 1.5 倍行距 | 18 磅 | 约 0.3 磅 | 0 | 约 28 磅 |
| 声明签名 | 1.5 倍行距 | 18 磅 | 0 | 6 磅 | 约 228 磅 |
| 声明日期 | 1.5 倍行距 | 18 磅 | 0 | 6 磅 | 约 257.5 磅 |
| 授权书标题 | 1.5 倍行距 | 18 磅 | 约 7.8 磅 | 约 7.8 磅 | 0 |
| 授权书正文 | 1.5 倍行距 | 18 磅 | 0 | 0 | 约 28 磅 |

自动化注意：对空段也必须写入段落标记字体字号，即 `w:pPr/w:rPr/w:sz` 与 `w:szCs`。例如声明正文后的空段为四号 14 pt；若只创建空段不设置段落标记，Word 会回退到 Normal 样式，签名区会整体上移。

声明页注意事项：

- 签名一般留待打印后手写。
- 日期是否填写按老师或学院要求处理。
- 不得删除声明页和授权书固定文字。
- 不得把声明页正文改成论文正文的固定 23 磅行距。
- 不得让中文题目、中文摘要或关键词出现在第 3 页。
- 声明页结束后必须使用分页符或下一页分节符进入中文摘要页。
- 如果有 `范本.docx`，应以范本第 49 到 63 段为直接参考，保持标题、正文、签名区、保密选项和授权签名区的相对顺序。

## 8. 摘要、关键词、目录格式规则

### 8.1 中文摘要页

| 项目 | 参数 |
|---|---|
| 页序 | 中文摘要必须单独从第 4 页开始，不与声明页同页 |
| 页码 | 从中文摘要页开始使用大写罗马数字页码，页脚居中 |
| 页首空段 | 中文题目前保留两个空段，固定 23 磅行距 |
| 中文论文题目 | 黑体小二号，居中，固定 23 磅行距 |
| 题目后空段 | 中文题目和摘要标题之间保留一个空段 |
| `摘　要` | 黑体小二号，居中，单倍行距，优先使用中文全角空格 `　` 分隔两个字 |
| 摘要标题后空段 | `摘　要` 和摘要正文之间保留一个单倍行距空段 |
| 摘要正文 | 宋体小四号，英文数字 Times New Roman，小四，首行缩进 2 字符，固定 23 磅 |
| `关键词：` | 黑体小四号，顶格，不缩进，单倍行距 |
| 关键词正文 | 宋体小四号，英文数字 Times New Roman，小四 |
| 关键词间隔 | 中文分号 `；` |

摘要内容只保留通用约束：

- 中文摘要一般约 300 字，应多于 250 字，少于 1 页。
- 摘要中不要引用参考文献。
- 关键词 3 到 5 个。
- 摘要中写到的核心功能、方法或对象，必须在正文、实现、测试中有对应内容。
- 摘要正文应按语义拆为 2 到 3 段，不要把背景、方法、实现和测试结果压成一个超长段落。
- 中文摘要页结束后必须分页进入英文摘要页。

### 8.2 英文摘要页

| 项目 | 参数 |
|---|---|
| 页序 | 英文摘要必须在中文摘要后单独起页 |
| 页码 | 延续中文摘要的大写罗马数字页码 |
| 页首空段 | 英文题目前保留两个空段，固定 23 磅行距 |
| 英文论文题目 | Times New Roman 小二号，加粗，居中，固定 23 磅 |
| 题目后空段 | 英文题目和 `ABSTRACT` 之间保留一个单倍行距空段 |
| `ABSTRACT` | Times New Roman 小二号，加粗，居中，单倍行距 |
| 摘要标题后空段 | `ABSTRACT` 和英文摘要正文之间保留一个空段 |
| 英文摘要正文 | Times New Roman 小四号，首行缩进 2 字符，固定 23 磅 |
| `Key words:` | Times New Roman 小四号，加粗，顶格，1.5 倍行距 |
| 英文关键词 | Times New Roman 小四号 |
| 关键词间隔 | 半角分号和空格，如 `System design; MySQL; Spring Boot` |

英文摘要规则：

- 英文摘要应在中文摘要定稿后更新。
- 英文摘要内容应与中文摘要一致。
- Key words 如换行，第二行应与上一行第一个关键词首字母对齐。
- 英文专有名词大小写必须全文统一。
- 英文摘要正文应按语义拆为 2 到 3 段，不要生成单个超长段落。
- 英文摘要结束后必须分页进入目录页，目录不得和英文关键词同页。

### 8.3 目录页

目录页必须同时执行本目录下的 `目录页可执行规范.md`。目录不能用普通正文段落手写，也不能用 `……` 或空格模拟点线；必须使用 Word TOC 域或等效的 TOC 样式、右对齐制表位和点线前导符。

| 项目 | 参数 |
|---|---|
| `目 录` | 黑体小二号，居中，固定 23 磅，段前 0.5 行，段后 0.5 行 |
| 章标题、参考文献、附录、科研情况目录、致谢 | 中文黑体小四号，数字英文 Times New Roman 小四号 |
| 其他目录项 | 中文宋体小四号，数字英文 Times New Roman 小四号 |
| 行距 | 固定 23 磅 |
| 缩进 | 按模板，不得手工乱缩进 |

目录规则：

- 目录应由 Word 目录域生成或按模板格式准确生成。
- 目录页不设置正文页眉。
- 正文修改、标题修改、图表移动、页码变化后，定稿前必须更新目录。
- 目录层级必须与正文标题一致。
- 范本式目录通常至少覆盖 1、1.1、1.1.1 三级标题；如果正文标题层级不足，必须先补正文标题，再更新目录。
- 目录条目不得继承正文段后 10 磅，不得使用普通正文 `Normal` 样式。
- 页码必须统一靠右，点线必须由制表位前导符生成，不得手写。

## 9. 正文与各后置部分格式规则

### 9.1 正文基本规则

正文从 `1 绪论` 或 `1 引言` 开始。正文页眉从此处开始，正文页码从阿拉伯数字 1 开始。

正文统一使用：

- 中文宋体小四号。
- 英文和数字 Times New Roman。
- 固定 23 磅行距。
- 首行缩进 2 字符。
- 段前段后 0。
- 两端对齐。

正文中除致谢外，不要出现“我、你、我们、你们”等第一或第二人称表达。可使用“本文”“本系统”“该模块”“用户”“管理员”等表述。

### 9.2 参考文献标题与正文

| 项目 | 参数 |
|---|---|
| `参考文献` 标题 | 黑体小二号，居中，段前 0.8 行，段后 0.5 行，固定 23 磅 |
| 文献条目 | 中文宋体小四号，英文数字 Times New Roman 小四号 |
| 行距 | 固定 23 磅 |
| 缩进 | 悬挂缩进 |
| 段前段后 | 0 |

悬挂缩进经验值：

| 条目编号 | 建议悬挂缩进 |
|---|---|
| `[1]` 到 `[9]` | 约 1.75 字符 |
| `[10]` 及以后 | 约 2.2 字符 |

### 9.3 附录

| 项目 | 参数 |
|---|---|
| `附 录` 标题 | 黑体小二号，居中，段前 0.8 行，段后 0.5 行，固定 23 磅 |
| 附录正文 | 中文宋体小四号，英文数字 Times New Roman 小四号，固定 23 磅 |

附录可以放核心代码截图、补充数据、较长表格等。若附录不需要，可不设置该部分。

### 9.4 科研情况目录

| 项目 | 参数 |
|---|---|
| 标题 | 黑体小二号，居中，段前 0.8 行，段后 0.5 行，固定 23 磅 |
| 条目 | 宋体小四号，英文数字 Times New Roman 小四号，固定 23 磅 |

无科研成果时可省略，除非学院或老师要求保留。

### 9.5 致谢

| 项目 | 参数 |
|---|---|

## 10. 图的格式与对象级规则

### 10.1 图对象规则

| 项目 | 要求 |
|---|---|
| 环绕方式 | 嵌入型，禁止浮动 |
| 图片段落对齐 | 居中 |
| 图片段落首行缩进 | 0 |
| 图片段落左右缩进 | 0 |
| 图片段落行距 | 单倍或 1.5 倍，优先保证图片完整显示 |
| 图片段落段前段后 | 0 |
| 图片最大宽度 | 不超过版心宽度 |

重点：图片段落不得套用正文“固定 23 磅 + 首行缩进 2 字符”的参数，否则图片容易被裁切、显示不全或位置异常。

### 10.2 图题规则

| 项目 | 参数 |
|---|---|
| 位置 | 图下方 |
| 编号 | 按章编号，如 `图3-1 系统框架图` |
| 序号与标题 | 序号后空一格 |
| 中文字体 | 黑体小四号 |
| 数字英文 | Times New Roman 加粗小四号 |
| 对齐 | 居中 |
| 首行缩进 | 0 |
| 段前 | 0.5 行，约 11.5 pt |
| 段后 | 0.5 行，约 11.5 pt |
| 行距 | 固定 23 磅 |

### 10.3 图文关系

- 每个图前必须有一段介绍该图的文字。
- 介绍文字应说明图展示什么对象、流程、界面或结构。
- 图和图题应尽量保持在同一页。
- 当页空间不足时，应将图整体移动到下一页，不得让图题留在上一页。
- 自动生成 `.docx` 时，图片段落应设置 `keep_with_next=True` 与 `keep_together=True`，图题段落应设置 `keep_together=True`；生成后仍必须导出 PDF/PNG 视觉确认。
- 代码截图类图片可以采用 IDE/编辑器截图风格，但必须保证所有字符串正常渲染；若代码片段含中文，应使用支持中文的字体回退，不得出现 `????`、缺字方框或乱码。
- 自动生成代码图时，应输出 `code_figures_manifest.json`，记录每张代码图的源文本和字体路径，并运行 `scripts/verify_code_figure_text.py code_figures_manifest.json` 检查非 ASCII 字符字体覆盖。

### 10.4 技术图样式

所有流程图、E-R 图、实体属性图、功能模块图、用例图、部署图、接口调用图和 AI 预测流程图应满足：

- 纯白背景，不使用灰色填充。
- 黑色文字。
- 黑色线条和箭头。
- 不得使用第二种颜色，包括低饱和彩色、主题色、渐变色、浅灰底和阴影。
- 不得黑底。
- 不得有网格背景。
- 同一篇论文内图形字体保持一致，优先使用宋体或与参考图等效的中文字体。
- 字体清晰可读，文字不得越出文本框。
- 线条清晰。
- 与图题编号对应。

功能模块图额外要求：

- 模块图连线通常不要使用箭头。
- 第二级模块可按前台/后台、角色或功能划分，但全篇逻辑要统一。

E-R 图额外要求：

- 总体 E-R 图可只展示实体和联系，不必画所有属性，但必须覆盖源码证据中的全部系统主题。
- 物理表、逻辑主题和业务过程可以同时出现，但必须在正文中说明哪些是数据库表，哪些是由接口、统计、AI 或部署逻辑形成的逻辑主题。
- 实体属性图应与后续数据表字段对应。
- E-R 图实体、属性、关系不得与数据表矛盾。
- 总体 E-R 图必须像参考图一样保持线条清楚：实体框、主题框和关系菱形之间要留出可见连接线，不能让关系菱形贴住实体框。
- 每一条关系应使用独立连接端口；禁止多条关系共用同一端口形成扇形乱线。
- 关系线不得穿过任何实体框、主题框、关系菱形或文字标记；`1`、`n` 等基数标记必须放在连接线附近的空白处，不能落入实体框或菱形。
- 优先按行列网格布置实体和关系，必要的斜线可以保留，但必须避免交叉、重叠、压字和穿框。
- 生成脚本应同时输出 `er_layout.json`，记录 boxes、relations、diamond_box、segments 和 cardinality_boxes；插入论文前运行 `scripts/verify_er_layout_geometry.py er_layout.json`，并以导出的 PDF/PNG 截图做最终视觉确认。

### 11.5 数据库设计表推荐列宽

对于常见 6 列数据库字段表，推荐列宽：

| 列 | 内容 | 推荐宽度 |
|---|---|---|
| 1 | 字段名 | 约 3.0 cm，必须容纳 `password_hash`、`estimated_minutes` 等长字段名 |
| 2 | 数据类型 | 约 2.3 cm |
| 3 | 长度 | 约 1.5 cm |
| 4 | 是否为空 | 约 2.1 cm |
| 5 | 约束 | 约 2.2 cm |
| 6 | 说明 | 约 4.3 cm |

数据库表注意：

- 字段名使用小写英文单词或下划线组合，不使用拼音。

- 整数型、逻辑型、日期型、枚举型字段通常不写长度。
- `varchar` 长度按实际需要设置，不得全部写成 `255`。
- 不同字段类型和长度应合理区分。
- 主键、外键、索引、默认值等应写清楚。

### 11.6 测试用例表推荐列宽

对于常见 5 列测试用例表，推荐列宽：

| 列 | 内容 | 推荐宽度 |
|---|---|---|
| 1 | 编号 | 1.85 cm |
| 2 | 测试功能 | 3.15 cm |
| 3 | 测试操作 / 输入 | 4.25 cm |
| 4 | 预期结果 / 实际结果 | 3.9 cm |
| 5 | 结论 | 1.45 cm |

测试表禁止事项：

- 不得所有实际结果都写“与预期一致”。
- 不得所有测试结果都写“页面更新成功”。
- 每行测试操作应有差异。
- 预期结果和实际结果应写成结果，而不是操作步骤。

## 12. 公式、脚注与代码截图

## 13. 图表编号与交叉引用

### 13.1 编号格式

| 对象 | 编号示例 |
|---|---|
| 图 | `图3-1 系统框架图` |
| 表 | `表3-1 用户信息表` |
| 公式 | `（3.1）` |

编号规则：

- 图、表按章单独编号。
- 序号后空一格再写标题。
- 全文编号格式必须统一。
- 不得同时出现 `图3-1` 和 `图3.1` 两种风格，除非老师明确要求保留模板点号格式。

### 13.2 正文引用

正文中应先介绍，再插入图表。常用写法：

- `系统总体结构如图3-1所示。`
- `用户信息表设计如表3-2所示。`
- `其计算方式如式（4.1）所示。`

禁止：

- 图表未在正文中引用。
- 先出现图表，后解释图表。
- 图题、表题编号与正文引用不一致。
- 图表编号跳号或重复。

## 14. 参考文献与正文引用格式

### 14.1 参考文献数量与来源

- 参考文献不少于 20 篇。
- 原则上以近 3 年内文献为主。
- 定稿时优先保证近 5 年文献占比较高。
- 必要的经典文献可以保留，但不能过多。
- 参考文献应与正文引用一一对应。

### 14.2 正文引用

| 项目 | 要求 |
|---|---|
| 编号 | 使用方括号，如 `[1]` |
| 字体 | Times New Roman 小四号 |
| 位置 | 右上角标 |
| 标点关系 | 放在句末标点前 |
| 链接 | 不得做成超链接 |
| 顺序 | 按正文首次出现顺序编号 |

标题、摘要、结论或总结中不要引用参考文献。

### 14.3 文献列表格式

常见类型格式如下：

| 类型 | 格式 |
|---|---|
| 期刊 | `[序号] 作者. 题名[J]. 刊名, 年份, 卷(期): 起页码-止页码.` |
| 图书 | `[序号] 作者. 书名[M]. 出版地: 出版社, 出版年份: 起页码-止页码.` |
| 学位论文 | `[序号] 作者. 题名[D]. 出版地: 单位, 年份.` |
| 会议论文 | `[序号] 作者. 题名[C]. 出版地: 出版者, 年份: 起页码-止页码.` |
| 报纸 | `[序号] 作者. 题名[N]. 报纸名, 出版日期(版次).` |
| 报告 | `[序号] 作者. 题名[R]. 报告地: 主办单位, 年份.` |
| 电子文献 | `[序号] 作者. 题名[EB/OL]. 地址, 发表或更新日期/引用日期.` |
| 专利 | `[序号] 专利所有者. 题名[P]. 国别: 专利号, 出版日期.` |
| 标准 | `[序号] 标准编号, 标准名称[S]. 出版地: 出版者, 年份.` |

### 14.4 作者规则

- 作者 3 人以内全部列出。
- 超过 3 人一般列前 3 名，后加“等”。
- 中文作者按中文姓名习惯书写。
- 外文姓名按国际惯例缩写，可省略缩写点，姓名之间空一格。
- 英文书名、期刊名可按要求使用斜体。

## 15. 内容写作的最低通用约束

本指南不是内容写作方案，但 Codex 生成初稿时必须遵守以下底线：

| 位置 | 最低约束 |
|---|---|
| 题目 | 中文题目必须与学校上报题目完全一致 |
| 摘要 | 不引用参考文献，功能与正文保持一致 |
| 绪论 | 不与摘要雷同，研究现状应有文献支撑 |
| 技术介绍 | 不得大段堆砌百科式技术背景 |
| 需求分析 | 角色、功能、技术需求应与设计、实现、测试对应 |
| 系统设计 | 模块图、流程图、E-R 图、数据表应逻辑一致 |
| 系统实现 | 重点写功能如何实现，不只写如何操作 |
| 系统测试 | 核心功能应有测试用例表和结果图 |
| 结论 | 总结工作、不足和改进，不写个人感想 |
| 致谢 | 可写个人表达，但注意加密稿风险 |

全文禁止：

- 编造不存在的功能、算法、数据表、测试结果。
- 摘要写了功能但实现章和测试章没有对应。
- 设计章有模块但实现章没有实现。
- 测试章只测试登录、注册等基础功能而缺少核心业务功能。
- 除致谢外出现“我、你、我们、你们”等不规范代词。

## 16. Codex 生成 Word 初稿的执行流程

### 16.1 推荐流程

1. 读取 `AGENTS.md`，确认本项目要求使用 Superpowers plugin skills。
2. 读取学校正式格式文件、初稿要求、范本、图表样式参考和已有修改指南。
3. 读取 `论文生成总流程清单.md`，先建立资料清单和证据台账。
4. 明确论文题目、专业、学生信息、指导教师、日期等封面字段。
5. 分析任务书、开题报告、源码目录、系统功能截图和图表样式参考，确定可写入论文的事实。
6. 按证据台账确定章节结构、图表清单、测试用例和参考文献需求。
7. 复制或创建符合学校模板结构的 Word 文档。
8. 按 `封面前两页可执行规范.md` 生成或复制中文封面和英文封面。
9. 按 `目录页可执行规范.md` 规划正文标题层级和目录页。
10. 按 `自动化脚本原理与复用规范.md` 决定是否使用 Word COM、OpenXML 或模板复制脚本；需要脚本时优先使用 `scripts` 文件夹中的参数化版本。
11. 建立分节：封面、声明、摘要、目录、正文、参考文献、附录、致谢。
12. 设置页面尺寸、页边距、页眉页脚距离。
13. 设置摘要和目录页码为大写罗马数字。
14. 设置正文从绪论开始阿拉伯页码从 1 起。
15. 设置正文页眉内容。
16. 生成标题结构和目录域。
17. 写入正文内容，整篇论文非空白可见字符数必须不少于 20000；初稿建议按 22000-25000 字组织，避免删改后低于学校要求。
18. 插入图、表、公式、代码截图并应用对象级格式。
19. 统一图题、表题、参考文献、正文引用。
20. 更新目录。
21. 对 Word 更新后的目录执行 OpenXML 格式固化，防止 TOC 缩进、行距和制表位回退。
22. 运行 Word / OpenXML 验证。
23. 导出 PDF 并检查封面、目录、图表、流程图、E-R 图、页眉页码。
24. 输出 Word 文件、PDF 文件和修改说明。

### 16.2 对象级处理顺序

自动化修改时，按以下对象分开处理：

| 顺序 | 对象 | 原因 |
|---|---|---|
| 1 | 文档结构与分节 | 决定页眉页码和目录 |
| 2 | 标题层级 | 决定目录和章节编号 |
| 3 | 正文段落 | 应用正文格式 |
| 4 | 图片段落 | 避免被正文固定行距裁切 |
| 5 | 图题和表题 | 避免继承正文缩进 |
| 6 | 表格和单元格 | 避免字段折行、底线缺失 |
| 7 | 参考文献 | 应用悬挂缩进 |
| 8 | 目录和页码 | 最后更新 |

### 16.3 禁止的全局操作

- 不得 `Ctrl+A` 后一键套用正文段落格式。
- 不得把所有段落都设置为首行缩进 2 字符。
- 不得把所有段落都设置为固定 23 磅，图片段落除外。
- 不得把表格单元格当正文段落处理。
- 不得把图题、表题当正文处理。
- 不得删除 Word 域代码后手写目录和页码。

## 17. 高风险错误禁止清单

### 17.1 页面与页码错误

- 摘要、目录出现正文页眉。
- 正文页码没有从 1 开始。
- 封面或声明页显示正文页码。
- 参考文献或致谢页码断裂。
- 页眉文字不是“武汉东湖学院本科生毕业论文（设计）”。
- 页边距被不同节覆盖成默认值。

### 17.2 标题与正文错误

- 章标题不是黑体小二号居中。
- 一级节标题不是黑体四号顶格。
- 二级节标题、三级标题格式混乱。
- 标题编号与目录不一致。
- 正文没有首行缩进 2 字符。
- 正文英文和数字不是 Times New Roman。

### 17.3 图片错误

- 图片被设置成浮动环绕。
- 图片段落被套固定 23 磅导致裁切。
- 图片段落继承正文首行缩进。
- 图题与图片跨页分离。
- 流程图、E-R 图有网格背景或黑底。
- PDF 中图中文字乱码、裁切或不可读。

### 17.4 表格错误

- 表格不是三线表。
- 表格缺少底边框。
- 单元格继承正文首行缩进。
- 字段名、`NULL`、`255`、`create_time` 等被无意义拆行。
- 表题和表格跨页分离。
- 表格文字不是五号或不是单倍行距。

### 17.5 参考文献错误

- 正文引用顺序与文献列表顺序不一致。
- 引用编号做成超链接。
- 引用编号位置不在标点前。
- 参考文献不足 20 篇。
- 文献条目没有悬挂缩进。
- 期刊文献缺少年份、卷期或页码。
- 标题、摘要、结论中出现参考文献引用。

## 20. Codex 执行摘要

在任何武汉东湖学院本科毕业论文 Word 生成任务中，Codex 必须把本指南作为格式硬约束执行。最重要的原则是：
