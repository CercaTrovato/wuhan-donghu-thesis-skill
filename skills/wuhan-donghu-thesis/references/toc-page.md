# 武汉东湖学院毕业论文目录页可执行规范

本文件用于约束 agent 生成、更新和验收武汉东湖学院本科毕业论文目录页。目录页不能用普通正文段落手工拼接，必须使用 Word 目录域或等效的 TOC 样式、制表位和点线前导符。

## 1. 核心原则

1. 目录页必须位于英文摘要之后、正文之前。
2. 目录标题为 `目　录`，中间使用全角空格或与范本一致的空格效果，居中。
3. 目录条目不得手写为 `标题……页码`。
4. 目录条目必须使用右对齐制表位和点线前导符，使页码统一靠右。
5. 目录内容必须从正文标题样式或 Word TOC 域生成，正文标题变动后必须更新目录。
6. 正文标题结构不足时，必须先补正文标题层级，再更新目录，不能只在目录页伪造三级条目。

## 2. 范本目录的关键特征

对照 `范本.docx`，目录页具有以下可执行特征：

| 项目 | 范本特征 | 执行要求 |
|---|---|---|
| 目录标题 | `目　录`，居中 | 不写成普通左对齐标题 |
| 条目样式 | `TOC 1`、`TOC 2` 等目录样式 | 不使用 `Normal` 或正文样式 |
| 条目行距 | 固定 23 磅 | 不使用单倍或默认行距 |
| 条目段前段后 | 0 | 不保留段后 10 磅 |
| 页码位置 | 右侧统一靠齐 | 使用右对齐制表位 |
| 点线 | 从标题后延伸到页码前 | 使用制表位前导符，不手写省略号 |
| 目录层级 | 至少覆盖 1、1.1、1.1.1 三级标题 | 与正文标题层级一致 |
| 页码 | 来自 Word 更新结果 | 不手工写死 |

## 3. 目录标题规范

| 项目 | 参数 |
|---|---|
| 文本 | `目　录` |
| 中文字体 | 黑体 |
| 字号 | 小二号，18 pt |
| 对齐 | 居中 |
| 行距 | 固定 23 磅 |
| 段前 | 约 0.5 行或按范本 |
| 段后 | 约 0.5 行或按范本 |
| 首行缩进 | 0 |

若从范本复制目录标题，应保留其直接段落格式。

## 4. 目录条目样式规范

目录条目必须使用 Word 目录样式或等效格式。推荐参数如下：

| 样式 | 适用条目 | 字体 | 字号 | 对齐 | 行距 | 段前 | 段后 | 缩进 |
|---|---|---|---:|---|---:|---:|---:|---|
| `TOC 1` | 章标题、参考文献、致谢 | 宋体/Times New Roman，章名可保留黑体效果 | 12 pt | 两端对齐 | 23 pt | 0 | 0 | 0 |
| `TOC 2` | 一级节、二级节、三级节 | 宋体/Times New Roman | 12 pt | 两端对齐 | 23 pt | 0 | 0 | 0 |
| `TOC 3` | 如工具必须分三级样式时使用 | 宋体/Times New Roman | 12 pt | 两端对齐 | 23 pt | 0 | 0 | 0 |

说明：范本中大量条目视觉上都落在 `TOC 2` 样式，但目录仍包含三级标题。若自动化工具能稳定区分 `TOC 1/TOC 2/TOC 3`，允许按 Word 标准样式区分；最终以视觉接近范本为准。

## 5. 制表位和点线规范

目录条目的页码必须通过制表位实现。

| 项目 | 参数 |
|---|---|
| 制表位位置 | 约 425.2 pt，或按当前页面版心右边界自动换算 |
| 制表位对齐 | 右对齐 |
| 前导符 | 点线前导符 |
| 条目文本结构 | `{标题文本}` + Tab + `{页码}` |
| 禁止结构 | `{标题文本}……{页码}` |

如果页面边距或版心变化，制表位应重新计算到版心右侧，不得停在页面中部。

## 6. 正文标题层级要求

目录是否接近范本，首先取决于正文标题层级。生成目录前必须检查正文 outline：

| 章节 | 范本式目录应包含的层级 |
|---|---|
| 第 1 章 绪论 | `1`、`1.1`、`1.2`、`1.3` |
| 第 2 章 需求分析与实施方案 | `2`、`2.1`、`2.1.1`、`2.1.2`、`2.2`、`2.2.1` 等 |
| 第 3 章 系统设计 | `3`、`3.1`、`3.1.1`、`3.1.2`、`3.2`、`3.3.1`、`3.3.2` 等 |
| 第 4 章 系统实现 | `4`、`4.1`、`4.2`、`4.2.1` 等按功能细分 |
| 第 5 章 系统测试 | `5`、`5.1`、`5.2`、`5.3`、`5.3.1` 等 |
| 第 6 章 结论 | `6`、`6.1`、`6.2` |

如果正文只有一级、二级标题，目录页会明显比范本稀疏。修复时必须先补正文三级标题，再更新目录。

## 7. 推荐生成流程

### 7.1 有范本时

1. 从 `范本.docx` 复制目录标题、TOC 样式定义和制表位参数。
2. 保留目标论文正文标题样式，并确保标题层级正确。
3. 删除目标论文原有手写目录段落。
4. 插入 Word TOC 域，目录级别覆盖 1-3 级标题。
5. 设置 `TOC 1/TOC 2/TOC 3` 样式的字体、字号、行距、段前段后和制表位。
6. 在 Word 中更新目录域。
7. 如果 Word 更新目录后又写入默认缩进或默认行距，按 `自动化脚本原理与复用规范.md` 执行 OpenXML 后处理，重新固化 `TOC1/TOC2/TOC3` 的缩进、行距和制表位。
8. 导出 PDF 或 PNG，与范本目录页视觉核对。

### 7.2 无范本时

1. 创建目录标题 `目　录`。
2. 插入 Word TOC 域，覆盖 1-3 级标题。
3. 设置目录条目为 12 pt、固定 23 磅、段前段后 0。
4. 设置右对齐制表位和点线前导符。
5. 更新目录域并检查页码。
6. 必要时用 OpenXML 后处理清除 Word 自动缩进，确保目录左边界、点线和页码右对齐稳定。

## 8. 禁止事项

- 不得用普通正文段落手写目录。
- 不得手写 `……`、`.....` 或空格模拟点线。
- 不得手工写死页码。
- 不得只在目录页添加三级条目，而正文没有对应标题。
- 不得让目录条目继承正文段后 10 磅。
- 不得让页码停在页面中部。
- 不得在正文修改后不更新目录。
- 不得让目录页显示正文页眉，除非学校模板明确要求。

## 9. 验收检查

生成或修改目录后必须检查：

1. `officecli view <docx> outline` 显示正文标题层级至少包含必要的三级标题。
2. 目录页段落样式不是 `Normal` 或普通正文样式。
3. 目录条目文本中没有硬编码的 `……`。
4. 目录条目包含 Tab，页码通过右对齐制表位靠右。
5. 点线从标题后延伸到页码前，长度随标题变化。
6. 目录至少覆盖正文、参考文献、致谢。
7. 导出 PDF/PNG 后与范本目录页相比，标题位置、点线、页码右对齐和行距基本一致。
8. 文档通过 Word/OpenXML 校验。

## 10. 常见错误与修正

| 失败表现 | 根因 | 修正 |
|---|---|---|
| 点线很短，页码在中间 | 没有右对齐制表位 | 设置右侧制表位和点线前导符 |
| 目录页只有一级二级标题 | 正文没有三级标题 | 先补正文标题层级，再更新目录 |
| 目录条目很稀疏 | 继承正文段后 10 磅 | TOC 样式段前段后设为 0 |
| 二级、三级条目整体右移 | Word 更新目录后写入默认 TOC 缩进 | 更新目录后再用 OpenXML 清除 `TOC1/TOC2/TOC3` 缩进 |
| 章标题粗细不一致 | 手工加粗或样式混乱 | 使用 TOC 样式统一控制 |
| 页码不随正文变化 | 手写页码 | 使用 Word TOC 域并更新 |
| 目录显示正文样式 | 手工普通段落 | 删除手写目录，插入 TOC 域 |
